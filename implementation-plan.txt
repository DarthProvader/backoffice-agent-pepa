### Klíčové principy
- **Žádné custom tools pro data** — agent má Bash a sám si píše SQL dotazy,
  Python skripty pro vizualizace, cURL příkazy. Stačí mu znát DB schéma.
- **Skills pro dokumenty** — pre-built skills od Anthropic (xlsx, pptx, pdf, docx)
  řeší generování souborů. Agent je volá automaticky.
- **MCP servery pro integrace** — Calendar a Gmail jsou hotové open-source MCP.
- **Jeden handleMessage()** — web, Telegram i cron tasky volají stejnou funkci.
  Agent má vždy stejný kontext a schopnosti.

-- Klienti firmy
CREATE TABLE clients (
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL,            -- "Jan Novák"
  email TEXT,
  phone TEXT,                    -- "+420 XXX XXX XXX"
  source TEXT,                   -- "web","doporuceni","sreality","bezrealitky","telefon"
  status TEXT DEFAULT 'active',  -- "active","inactive","vip"
  notes TEXT,
  created_at DATE,
  updated_at DATE
);

-- Nemovitosti v portfoliu
CREATE TABLE properties (
  id INTEGER PRIMARY KEY,
  address TEXT NOT NULL,          -- "Osadní 12, Praha 7"
  city TEXT DEFAULT 'Praha',
  district TEXT,                  -- "Holešovice","Vinohrady","Karlín","Dejvice"
  type TEXT,                      -- "byt","komercni","dum","pozemek"
  disposition TEXT,               -- "2+kk","3+1","kancelar"
  area_m2 REAL,
  price REAL,                     -- v CZK
  status TEXT,                    -- "v_nabidce","rezervovano","prodano","stazeno"
  renovation_status TEXT,         -- "po_rekonstrukci","pred_rekonstrukci","castecna",NULL
  building_modifications TEXT,    -- popis úprav, NULL pokud chybí
  energy_rating TEXT,             -- "A"-"G", NULL pokud chybí
  description TEXT,
  listed_at DATE,
  sold_at DATE,
  owner_id INTEGER REFERENCES clients(id)
);

-- Leady (zájemci o nemovitost)
CREATE TABLE leads (
  id INTEGER PRIMARY KEY,
  client_id INTEGER REFERENCES clients(id),
  property_id INTEGER REFERENCES properties(id),
  status TEXT,                    -- "novy","kontaktovan","prohlidka","nabidka","uzavreno","ztracen"
  source TEXT,
  notes TEXT,
  created_at DATE,
  converted_at DATE
);

-- Prodeje
CREATE TABLE sales (
  id INTEGER PRIMARY KEY,
  property_id INTEGER REFERENCES properties(id),
  buyer_id INTEGER REFERENCES clients(id),
  seller_id INTEGER REFERENCES clients(id),
  price REAL,                     -- skutečná prodejní cena
  commission REAL,                -- provize firmy
  sold_at DATE,
  notes TEXT
);

-- Prohlídky nemovitostí
CREATE TABLE viewings (
  id INTEGER PRIMARY KEY,
  property_id INTEGER REFERENCES properties(id),
  client_id INTEGER REFERENCES clients(id),
  scheduled_at DATETIME,
  status TEXT,                    -- "planovana","probehnuta","zrusena"
  feedback TEXT,
  created_at DATE
);

-- Monitoring realitních serverů (výsledky scrapingu)
CREATE TABLE listing_snapshots (
  id INTEGER PRIMARY KEY,
  source TEXT,                    -- "sreality","bezrealitky"
  external_id TEXT,
  url TEXT,
  title TEXT,
  price REAL,
  area_m2 REAL,
  district TEXT,

- Pro scraping Sreality/Bezrealitky použij WebFetch
- Výsledky porovnávej s tabulkou listing_snapshots

## Scheduled tasky
Když uživatel požádá o opakovanou úlohu ("sleduj každé ráno...",
"každý pátek připrav report...", "připomeň mi v 15:00..."):

1. Vytvoř prompt soubor: /data/scheduled-tasks/{task-id}.md
   - Detailní instrukce co má task dělat
   - Kam uložit výsledky
   - Jak notifikovat uživatele
2. Zavolej API pro registraci tasku:
   ```bash
   curl -X POST http://localhost:3001/api/tasks \
     -H "Content-Type: application/json" \
     -d '{"id":"...","name":"...","cron":"0 7 * * *","prompt_file":"...","notify":["telegram","push"]}'
   ```
3. Pro připomínky (jednorázové) použij cron pro konkrétní datum/čas
4. Informuj uživatele o nastavení a frekvenci

Pro zobrazení existujících tasků:
```bash
curl http://localhost:3001/api/tasks
```

Pro smazání:
```bash
curl -X DELETE http://localhost:3001/api/tasks/{id}
```

## 5. Klíčové soubory k implementaci

### 5.1 server/src/agent.ts — jádro

```typescript
import { query, ClaudeAgentOptions } from "@anthropic-ai/claude-agent-sdk";

interface AgentMessage {
  type: "text" | "tool_use" | "tool_result" | "thinking" | "done";
  content: any;
}

export async function handleMessage(
  userMessage: string,
  onChunk: (msg: AgentMessage) => void
): Promise<void> {
  for await (const message of query({
    prompt: userMessage,
    options: {
      systemPrompt: {
        type: "preset",
        preset: "claude_code",
        append: fs.readFileSync("./system-prompt.md", "utf-8"),
      },
      cwd: "/data",
      settingSources: ["project"],   // načte .claude/skills/
      allowedTools: [
        "Bash", "Read", "Write", "Edit",
        "Glob", "Grep",
        "WebSearch", "WebFetch",
        "Skill",
      ],
      permissionMode: "bypassPermissions",
      mcpServers: {
        "google-calendar": {
          command: "npx",
          args: ["-y", "@cocal/google-calendar-mcp"],
        },
        "gmail": {
          command: "node",
          args: ["/app/mcp-servers/gmail/index.js"],
        },
      },
      maxTurns: 15,
      maxBudgetUsd: 0.50,   // safety limit per dotaz
    },
  })) {
    // Parsuj a forwarduj message type do callbacku
    onChunk(parseAgentMessage(message));
  }
}
```

### 5.2 server/src/telegram.ts — Telegram bot

```typescript
import { Bot } from "grammy";
import { handleMessage } from "./agent";

const bot = new Bot(process.env.TELEGRAM_BOT_TOKEN!);
const ALLOWED_USERS = process.env.TELEGRAM_ALLOWED_USERS?.split(",") || [];

bot.on("message:text", async (ctx) => {
  // Bezpečnost: jen povolení uživatelé
  if (!ALLOWED_USERS.includes(String(ctx.from?.id))) return;

  await ctx.reply("⏳ Pracuji na tom...");

  let fullResponse = "";
  await handleMessage(ctx.message.text, (chunk) => {
    if (chunk.type === "text") fullResponse += chunk.content;
  });

  // Telegram max 4096 znaků — rozdělit pokud delší
  const chunks = splitMessage(fullResponse, 4000);
  for (const chunk of chunks) {
    await ctx.reply(chunk, { parse_mode: "Markdown" });
  }

  // Pokud agent vygeneroval soubory, poslat jako dokumenty
  const files = findNewOutputFiles();
  for (const file of files) {
    await ctx.replyWithDocument(new InputFile(file.path), {
      caption: file.name,
    });
  }
});

export { bot };
```

### 5.3 server/src/scheduler.ts — Scheduled tasks

```typescript
import cron from "node-cron";
import fs from "fs";
import { handleMessage } from "./agent";
import { sendTelegramNotification } from "./notifications";
import { sendWebPush } from "./notifications";

interface ScheduledTask {
  id: string;
  name: string;
  cron: string;           // "0 7 * * *"
  promptFile: string;     // "/data/scheduled-tasks/morning-holesovice.md"
  notify: string[];       // ["telegram", "push", "email"]
  enabled: boolean;
  createdAt: string;
  lastRun?: string;
  lastResult?: string;
}

const activeJobs = new Map<string, cron.ScheduledTask>();

export function loadAndStartTasks() {
  const tasks: ScheduledTask[] = JSON.parse(
    fs.readFileSync("/data/scheduled-tasks/tasks.json", "utf-8")
  );
  for (const task of tasks) {
    if (task.enabled) registerTask(task);
  }
}

export function registerTask(task: ScheduledTask) {
  const job = cron.schedule(task.cron, async () => {
    const prompt = fs.readFileSync(task.promptFile, "utf-8");
    let result = "";

    await handleMessage(prompt, (chunk) => {
      if (chunk.type === "text") result += chunk.content;
    });

    // Ulož výsledek
    const resultFile = `/data/task-results/${task.id}-${Date.now()}.json`;
    fs.writeFileSync(resultFile, JSON.stringify({
      taskId: task.id, taskName: task.name,
      completedAt: new Date().toISOString(),
      summary: result,
    }));

    // Notifikace
    if (task.notify.includes("telegram")) {
      await sendTelegramNotification(`✅ ${task.name}\n\n${result}`);
    }
    if (task.notify.includes("push")) {
      await sendWebPush({ title: task.name, body: result.slice(0, 200) });
    }
  });

  activeJobs.set(task.id, job);
}

export function removeTask(taskId: string) { /* ... */ }
export function listTasks(): ScheduledTask[] { /* ... */ }
```

### 5.4 web/hooks/useAgentChat.ts — Frontend WebSocket hook

```typescript
import { useState, useCallback, useRef, useEffect } from "react";

interface Message {
  id: string;
  role: "user" | "agent";
  content: string;
  toolCalls?: ToolCall[];   // pro activity panel
  files?: FileRef[];         // pro download linky
  chart?: ChartData;         // pro inline graf
  timestamp: Date;
}

export function useAgentChat() {
  const [messages, setMessages] = useState<Message[]>([]);
  const [isLoading, setIsLoading] = useState(false);
  const wsRef = useRef<WebSocket | null>(null);

  useEffect(() => {
    const ws = new WebSocket(process.env.NEXT_PUBLIC_WS_URL!);
    ws.onmessage = (event) => {
      const data = JSON.parse(event.data);
      // Zpracuj streaming chunky:
      // - text → připoj k poslední agent zprávě
      // - tool_use → přidej do toolCalls
      // - done → setIsLoading(false)
    };
    wsRef.current = ws;
    return () => ws.close();
  }, []);

  const sendMessage = useCallback((text: string) => {
    setMessages((prev) => [...prev, { role: "user", content: text, ... }]);
    setIsLoading(true);
    wsRef.current?.send(JSON.stringify({ type: "chat", content: text }));
  }, []);

  return { messages, isLoading, sendMessage };
}
```

| Web Push | Pokud má uživatel povolené notif. | web-push knihovna + VAPID |
| In-app (WebSocket) | Pokud je web UI otevřené | ws.send() do připojených klientů |
| Email | Pokud uživatel nastavil | Gmail/Resend MCP přes agent |

### Flow po dokončení scheduled tasku:
1. Agent doběhne → výsledek uložen do `/data/task-results/`
2. Scheduler přečte výsledek
3. Pošle na Telegram (bot.api.sendMessage s formátovaným textem)
4. Pošle Web Push (web-push knihovna, payload s title + summary)
5. Pošle přes WebSocket do připojených web klientů
6. Volitelně: agent sám pošle email (pokud je v task.notify)

### Web Push setup:
- VAPID klíče: vygenerovat `web-push generate-vapid-keys`
- Frontend: service worker `sw.js` + `usePushNotifications` hook
- Backend: `POST /api/push/subscribe` uloží subscription, `sendWebPush()` odesílá
- Klik na notifikaci → otevře web UI s výsledky

## 8. Fallback plány

### Fallback A: OpenClaw (pokud Agent SDK nefunguje)

**Kdy přepnout:** Agent SDK query() je nestabilní, OOM issues, nepodporuje skills.
**Co se změní:**
- Runtime: OpenClaw Gateway místo vlastního Node.js serveru
- Billing: API key (pay-per-token) místo Max sub
- Telegram: OpenClaw nativní grammY adaptér (nemusíš psát bot)
- Config: SOUL.md + AGENTS.md místo CLAUDE.md
- Skills: kompatibilní (SKILL.md formát je stejný)
- Web UI: OpenClaw WebChat jako základ, nebo vlastní frontend přes Gateway WS
- Scheduled tasks: OpenClaw heartbeaty a proaktivní chování

### Fallback B: Claude API + tool-use (nejjednodušší)

**Kdy přepnout:** Agent SDK i OpenClaw selhávají, potřebuji rychlé řešení.
**Co se změní:**
- Runtime: přímo Anthropic Messages API s tools (ne Agent SDK)
- Musím definovat vlastní tools (query_database, generate_chart, ...)
- Agent nemá Bash — flexibilita omezená na předem definované tools
- Menší wow efekt, ale spolehlivé a předvídatelné
- Frontend zůstává stejný
